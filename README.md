# 역사 여행 계획 세우기 — 배포 안내 (교사용)

고등학교 한국사 활동 웹페이지입니다. 파일 3개(`index.html`, `style.css`, `app.js`)로만 동작하고,
교사용 QR 생성기(`qr.html`)가 함께 들어 있습니다.

---

## 0. 이 페이지가 어떻게 동작하나요? (중요)

- 학생 입력은 **각자 태블릿의 브라우저(localStorage)에만** 저장됩니다.
- **GitHub나 어떤 서버로도 전송되지 않습니다.** (코드에 `fetch`·폼 전송·외부 주소가 전혀 없습니다.)
- 맛집·방문지에 붙인 **사진도 태블릿 안에만** 저장됩니다(자동으로 작게 압축). 사진이 너무 많으면
  저장 공간이 부족할 수 있는데, 그때는 안내 메시지가 뜨고 페이지는 멈추지 않습니다.
- 제출은 완성본 화면의 **📄 PDF로 저장**으로 파일을 만든 뒤, **이메일(`k2armor@gw.hs.kr`)로 전송**합니다.
  완성본 화면에 이메일 주소·복사 버튼·발송 단계 안내가 표시됩니다. 학교 패드 정책상 자동 메일 연동은
  하지 않고, 학생이 **Gmail 앱 또는 gmail.com**에서 PDF를 첨부해 직접 보냅니다.
  (PDF에는 사진이 함께 인쇄되고, 파일명은 `학번_이름`으로 채워집니다.)
- 여러 반이 같은 URL을 써도 됩니다. **태블릿마다 저장 공간이 따로**라 서로 섞이지 않습니다.
  (단, 같은 태블릿을 여러 학생이 쓰면 앞 학생 내용이 남아 있을 수 있으니, 시작 전 브라우저에서
  기존 내용을 지우거나 다른 태블릿을 쓰게 하세요.)

---

## 1. GitHub 저장소 만들기

1. https://github.com 에 로그인 (교사 계정 하나면 충분).
2. 오른쪽 위 **＋ → New repository** 클릭.
3. **Repository name** 에 `history-trip` 처럼 영어로 입력.
4. **Public** 선택 → **Create repository**.

## 2. 파일 올리기

1. 만든 저장소 화면에서 **Add file → Upload files** 클릭.
2. 이 폴더의 파일을 끌어다 놓기: `index.html`, `style.css`, `app.js`, `qr.html`, `README.md`.
3. 아래 **Commit changes** 버튼 클릭.

## 3. GitHub Pages 켜기

1. 저장소 상단 **Settings** 탭 클릭.
2. 왼쪽 메뉴에서 **Pages** 클릭.
3. **Build and deployment → Source** 를 **Deploy from a branch** 로.
4. **Branch** 를 `main`, 폴더는 `/ (root)` 로 두고 **Save**.
5. 1~2분 뒤 새로고침하면 위쪽에 공개 주소가 나옵니다:
   `https://<내아이디>.github.io/history-trip/`

## 4. 공개 URL 확인

- 위 주소를 직접 열어 페이지가 뜨는지 확인하세요.
- 이 주소가 **학생들이 접속할 주소**입니다. (학생은 GitHub 계정·로그인·업로드 전부 필요 없음.)

---

## 5. QR 코드 만들기 (외부 서비스 없이)

1. 같은 주소 뒤에 `qr.html` 을 붙여 엽니다:
   `https://<내아이디>.github.io/history-trip/qr.html`
   (또는 내 컴퓨터에서 `qr.html` 파일을 더블클릭해 열어도 됩니다.)
2. 칸에 **4번의 공개 URL** 을 붙여넣고 **QR 만들기**.
3. **인쇄 / PDF** 로 출력해 교실에 붙이거나 화면에 띄우면, 학생들이 태블릿 카메라로 스캔해 들어옵니다.

> QR이 안 만들어질 때: 크롬(데스크톱)에서 활동 페이지를 연 뒤 주소창 우클릭 →
> "이 페이지의 QR 코드 생성" 으로도 오프라인 생성이 됩니다.

---

## 수업 흐름 요약

배포(교사, 1회) → QR 게시 → 학생이 태블릿으로 URL 접속 → 활동(자동저장) →
완성본에서 **PDF 저장 → Gmail로 `k2armor@gw.hs.kr` 에 첨부 전송**.
